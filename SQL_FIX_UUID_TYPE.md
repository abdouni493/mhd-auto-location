# SQL Migration Fix - UUID Type Correction

## The Issue
The original migration had a type mismatch:
- **cars** table: Uses UUID for `id` 
- **vehicle_expenses** table (old): Was using TEXT for `car_id`
- **Result**: Foreign key constraint error

## The Fix ✅
Both tables now use UUID consistently:
```sql
-- Cars table (existing)
cars.id → UUID

-- Updated tables
store_expenses.id → UUID
vehicle_expenses.id → UUID
vehicle_expenses.car_id → UUID (references cars.id)
```

## How to Apply the Fix

### Option 1: Drop and Recreate (Recommended for Development)
If you have test data only, drop and recreate:

```sql
-- Drop existing tables (this deletes all data!)
DROP TABLE IF EXISTS vehicle_expenses CASCADE;
DROP TABLE IF EXISTS store_expenses CASCADE;

-- Then run the updated expenses_migration.sql
```

### Option 2: Alter Existing Tables (for Production)
If you have production data to preserve:

```sql
-- Convert store_expenses.id from TEXT to UUID
ALTER TABLE store_expenses 
ALTER COLUMN id SET DATA TYPE uuid USING (id::uuid);

-- Convert vehicle_expenses
ALTER TABLE vehicle_expenses 
ALTER COLUMN id SET DATA TYPE uuid USING (id::uuid);

ALTER TABLE vehicle_expenses 
ALTER COLUMN car_id SET DATA TYPE uuid USING (car_id::uuid);

-- Drop and recreate the foreign key
ALTER TABLE vehicle_expenses 
DROP CONSTRAINT vehicle_expenses_car_id_fkey;

ALTER TABLE vehicle_expenses 
ADD CONSTRAINT vehicle_expenses_car_id_fkey 
FOREIGN KEY (car_id) REFERENCES cars(id) ON DELETE CASCADE;
```

## Steps to Complete

1. **Open Supabase SQL Editor**
   - Go to your Supabase dashboard
   - Navigate to SQL Editor tab

2. **Choose Your Path:**

   **If you have NO data** (development):
   - Copy the conversion script above (Option 1)
   - Paste into SQL Editor
   - Click "Run"

   **If you have data to keep** (production):
   - Copy the alteration script above (Option 2)
   - Paste into SQL Editor
   - Click "Run"

3. **Verify Tables Created**
   - Go to Table Editor
   - Confirm `store_expenses` and `vehicle_expenses` exist
   - Check that `id` columns show as "uuid" type

4. **Test the Application**
   - Navigate to Expenses module
   - Add a new vehicle expense
   - Click Save
   - Refresh page
   - ✅ Expense should persist with correct data
